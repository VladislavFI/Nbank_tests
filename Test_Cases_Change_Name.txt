Итерация 2: Изменение имени пользователя

Тестовый пользователь:

  {
  "username": "Filip_SM",
  "password": "verys111ETRongPassword33$",

}

Требования:
- не может быть пустым
- должно содержать от 3 до 15 символов
- может содержать только буквы, цифры, дефисы, подчёркивания и точки


Кейсы:

1. Изменение имени пользователя (позитивный)

curl -X 'PUT' \
  'http://localhost:4111/api/v1/customer/profile' \
  -H 'accept: */*' \
  -H 'Authorization: Basic RmlsaXBfU006dmVyeXMxMTFFVFJvbmdQYXNzd29yZDMzJA==' \
  -H 'Content-Type: application/json' \
  -d '{
  "name": "Ma_3-."
}'

Значения позитивных имен:
"Ma_3-."
"Maa",
"Maa3",
"Maa12345678912",
"Maa123456789123",
"Maa1234567891234",
"AAAAA",
"12345",
"-----",
"_____",
"....."

2. Изменение имени пользователя (негативный)

  Ma
  Maa12345678912344
  ""
